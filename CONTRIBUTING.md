## How to Contribute

### Reporting Bugs üêõ

If you find a bug, please open an issue with:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Browser and OS information
- Screenshots if applicable

### Suggesting Features üí°

Feature requests are welcome! Please include:
- Clear use case
- Expected behavior
- How it improves user experience
- Any mockups or examples

### Code Contributions üîß

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
   - Write clean, readable code
   - Follow existing code style
   - Add comments for complex logic
4. **Test thoroughly**
   - Test on multiple browsers
   - Check responsive design
   - Verify data persistence
5. **Commit with clear messages**
   ```bash
   git commit -m "feat: Add export to PDF functionality"
   ```
6. **Push and create a Pull Request**

## Commit Message Guidelines

Use conventional commits:
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation changes
- `style:` Formatting, missing semicolons, etc.
- `refactor:` Code restructuring
- `test:` Adding tests
- `chore:` Maintenance tasks

Examples:
```
feat: Add dark mode toggle
fix: Correct Nepali date conversion for leap years
docs: Update installation instructions
style: Format code with prettier
refactor: Simplify calendar rendering logic
```

## Code Style

- Use 2 spaces for indentation
- Use meaningful variable names
- Add JSDoc comments for functions
- Keep functions small and focused
- Avoid global variables when possible

## Testing Checklist

Before submitting a PR:
- [ ] Code runs without errors
- [ ] Tested on Chrome, Firefox, Safari
- [ ] Mobile responsive design verified
- [ ] LocalStorage operations work correctly
- [ ] No console errors or warnings
- [ ] Calendar navigation works properly
- [ ] Date selection and saving functional
- [ ] All existing features still work

## Need Help?

Feel free to:
- Open an issue for questions
- Ask for clarification on existing code
- Request code review guidance

## Code of Conduct

- Be respectful and constructive
- Welcome newcomers warmly
- Focus on the code, not the person
- Help others learn and grow
